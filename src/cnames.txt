To deploy public web APIs:
	Treat public web APIs as products. Create the role of API product manager to understand user requirements and communicate a clear roadmap.
	Plan a business strategy for your public web APIs. Carefully consider which developers and &&  by extension &&  which users will engage them. Also consider how they can be exploited through web &&  Internet of Things (IoT) &&  mobile and virtual personal assistants (such as Amazon Alexa or Microsoft Cortana). Understand that public web APIs are often an entry point into a larger product or service.
	Do not take it for granted that a public web API will be used. Establishing traction requires cultivating a community of developers &&  tapping into nearby communities &&  as well as rich documentation in a self-service developer portal.
	Ensure that an API has an effective security strategy. This can be a combination of application delivery controller infrastructure &&  web application firewalls and dedicated API gateways. Bot mitigation is also key &&  since public web APIs must be protected from misuse that can lead to data loss &&  fraud &&  and skewed analytics.
	Use full life cycle API management to manage the API through the entire process of planning &&  design &&  runtime deployment and versioning &&  through to retirement.
To consume public web APIs:
	Ensure that pricing is understood. API pricing may take the form of consumption-based pricing &&  tiered usage or subscriptions for example (see "Top 10 Things CIOs Need to Know About APIs and the API Economy" G00318859)
	Take steps to ensure that your organization is protected from availability &&  security &&  and privacy issues related to consumption of third-party APIs. This can be achieved with a cloud access security broker or via an API Gateway for outbound API consumption
